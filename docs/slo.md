# SLI/SLO Plan

## Service Level Indicators

The project tracks four demo SLIs:

- HTTP success rate.
- p95 latency.
- Availability.
- Error rate.

## Demo SLOs

| SLO | Target |
| --- | --- |
| Availability during normal operation | >= 99% in the demo test window |
| p95 latency under demo load | < 500 ms |
| Failure alert time | < 1 minute after simulated failure |
| Runtime security alert time | < 1 minute after suspicious container behavior |

## Measurement Approach

- Prometheus records HTTP and Kubernetes metrics.
- Grafana displays SLO panels.
- Alertmanager fires operational alerts.
- Falco records runtime security detections.
- Demo load can be generated by `k6`, `hey`, or a small script.

## Implemented Signals

| Signal | Source |
| --- | --- |
| HTTP success rate | `voting_http_requests_total`, `result_http_requests_total` |
| p95 latency | `voting_http_request_duration_seconds_bucket`, `result_http_request_duration_seconds_bucket` |
| Availability | Prometheus `up` for `vote` and `result` ServiceMonitor targets |
| Error rate | HTTP `5xx` request counters |

The Helm chart renders a `ServiceMonitor` and `PrometheusRule` when `monitoring.enabled=true`.
Grafana loads `observability/grafana-dashboards/voting-slo.json` through the dashboard sidecar.
