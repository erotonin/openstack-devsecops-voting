# SLI/SLO Plan

## Service Level Indicators

The project tracks four demo SLIs:

- HTTP success rate.
- p95 latency.
- Availability.
- Error rate.

## Demo SLOs

| SLO | Target |
| --- | --- |
| Availability during normal operation | >= 99% in the demo test window |
| p95 latency under demo load | < 500 ms |
| Failure alert time | < 1 minute after simulated failure |
| Runtime security alert time | < 1 minute after suspicious container behavior |

## Measurement Approach

- Prometheus records HTTP and Kubernetes metrics.
- Grafana displays SLO panels.
- Alertmanager fires operational alerts.
- Falco records runtime security detections.
- Demo load can be generated by `k6`, `hey`, or a small script.

## Report Evidence

Collect:

- Grafana SLO screenshot.
- Alertmanager alert timestamp.
- Falco event timestamp.
- Test start/end time.
- Calculated RTO/RPO during DR drill.
